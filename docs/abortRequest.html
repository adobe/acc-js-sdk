--- layout: page title: Handling abort requests ---

When building applications, we make API requests to servers for different reasons; to authenticate user credentials, fetch resources, create resources, and so much more. In JavaScript,we can abort an API request i.e cancel a request before it is completed. There are many reasons why you may want to do this. One of them is to prevent a request from being sent to the server if the user has already navigated away from the page the request was made. Canceling requests can help you avoid unnecessary network traffic, save memory usage and resources, and improve performance.

Aborting API requests in fetch

The fetch method is a JavaScript function used to make API requests. It takes two arguments: the URL of the API endpoint and an object containing the request options.

Here's the flow of how canceling a fetch call works:

  • Create an AbortController instance
  • That instance has a signal property
  • Pass the signal as a fetch option for signal
  • Call the AbortController's abort property to cancel all fetches that use that signal.

Aborting API requests in Axios

Axios is a JavaScript library that is used to make API requests. It has a very similar syntax to the fetch method. It also accepts a signal property in the request options object. When the aborted flag of the signal object is updated to true, axios expression is notified, and the request is canceled.

Passing AbortSignal via PushDown Mechanism

SDK provides the support to pass the signal directly to transport layer via PushDown Mechanism

client.NLWS.pushDown({ signal: instance.current.signal }).xtkQueryDef.create(queryDef)
